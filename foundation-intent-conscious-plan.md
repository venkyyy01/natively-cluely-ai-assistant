# Foundation Intent And Conscious Mode Plan

## Goal
Make the Foundation Models intent path measurably reliable in this repo, use calibrated confidence to improve acceleration and route selection, and tighten conscious-mode planning and verification without regressing the existing legacy fallback or Windows behavior.

## Tasks
- [ ] Build a labeled intent eval corpus and runner in `electron/evals/intentClassificationEval.ts` with fixtures covering all 8 current intents plus ambiguous negatives and transcript-dependent follow-ups. → Verify: running the eval prints per-intent accuracy, confusion matrix, provider split, fallback rate, and confidence buckets.
- [ ] Calibrate Foundation intent confidence in `applesilicon/macos-foundation-intent-helper/Sources/main.swift`, `electron/llm/providers/FoundationModelsIntentProvider.ts`, and `electron/llm/providers/IntentClassificationCoordinator.ts` so low-confidence or contradictory Foundation predictions fall back to legacy before they reach answer generation. → Verify: targeted tests cover low-confidence fallback, contradiction fallback, retry exhaustion, and unchanged Windows behavior.
- [ ] Add confidence-quality instrumentation in `electron/IntelligenceEngine.ts` and `electron/latency/AnswerLatencyTracker.ts` for `intentConfidence`, `intentProviderUsed`, `intentRetryCount`, `intentFallbackReason`, `prefetchedIntentUsed`, and route outcome metadata. → Verify: latency snapshots include the new fields and never log raw transcript text.
- [ ] Tighten acceleration prefetch in `electron/conscious/ConsciousAccelerationOrchestrator.ts` and `electron/services/AccelerationManager.ts` so prefetched intent is cached with revision-aware freshness and only unlocks speculative answer work when confidence is strong enough to justify the spend. → Verify: tests prove stale prefetched intents are discarded and speculative answer startup is skipped for weak/general intent signals.
- [ ] Thread intent confidence into the conscious path via `electron/conscious/ConsciousIntentService.ts`, `electron/conscious/ConsciousPreparationCoordinator.ts`, `electron/conscious/ConsciousOrchestrator.ts`, and `electron/conscious/ConsciousAnswerPlanner.ts` so conscious mode is favored for strong behavioral/system-design/live-coding signals and downgraded for weak or uncertain routing. → Verify: route tests show deterministic downgrade from `conscious_answer` to standard routes on weak signals and stable continuation on strong thread-compatible signals.
- [ ] Expand conscious replay and verifier coverage in `electron/conscious/ConsciousEvalHarness.ts`, `electron/conscious/ConsciousVerifier.ts`, `electron/conscious/ConsciousProvenanceVerifier.ts`, `electron/conscious/QuestionReactionClassifier.ts`, and `electron/conscious/AnswerHypothesisStore.ts` to include behavioral STAR depth, topic shifts, unsupported metric claims, unsupported technology claims, and coding/system-design continuation cases. → Verify: harness summaries report pass/fail by scenario family and catch known bad continuations.
- [ ] Prototype a persistent Foundation helper session only if the new latency measurements show helper startup dominates p95; otherwise keep the current CLI model and focus on prompt/schema efficiency. Target files: `applesilicon/macos-foundation-intent-helper/Sources/main.swift` and the TS helper client path. → Verify: benchmark comparison shows whether session reuse materially reduces p95 and first-token latency before we accept the added complexity.
- [ ] Re-run the full verification stack: `npm run typecheck`, targeted intent/coordinator tests, conscious harnesses, and the intent eval runner, then compare Foundation-only, legacy-only, and coordinated results before rollout. → Verify: the coordinated path beats or matches the legacy baseline on the labeled set, confidence calibration is reported, and conscious-mode regressions are caught before shipping.

## Done When
- [ ] We can measure intent quality in-repo instead of relying on anecdotes.
- [ ] Coordinated intent classification uses Foundation first when eligible, but confidence-aware fallback keeps combined quality at or above the current legacy baseline.
- [ ] Acceleration only spends speculative work on fresh, high-value intent predictions.
- [ ] Conscious mode gets stronger routing, stronger replay/verifier coverage, and cleaner fallback behavior on uncertain turns.
